# Advent of Code 2025 - Development Patterns

## Project Overview
This is a Vue 3 + TypeScript project for solving Advent of Code 2025 challenges with a visual interface.

## Development Workflow for New Days

When starting a new day (e.g., Day 3, Day 4, etc.), follow this workflow:

### 1. Create solve.ts file
- Create `src/views/dayN/solve.ts` with the solution logic
- Export `solvePart1()` and `solvePart2()` functions
- Keep functions pure - they should accept input as a string parameter and return the result
- Example structure:
```typescript
export function solvePart1(input: string): number {
  // Solution logic
}

export function solvePart2(input: string): number {
  // Solution logic
}
```

### 2. Write tests
- Create `src/views/dayN/solve.test.ts` 
- Write tests using the example inputs from the Advent of Code problem
- Test both parts of the puzzle
- Ensure all tests pass before moving to UI

### 3. Wire up the UI
- Create or update `src/views/dayN/DayN.vue`
- Use the `useInputLoader` composable to load input files
- Import and call `solvePart1()` and `solvePart2()` from `solve.ts`
- Display results in the UI
- **IMPORTANT**: Display answers as raw numbers without formatting (no commas, no `.toLocaleString()`)

## File Structure
```
src/
  views/
    dayN/
      solve.ts          # Pure solution functions
      solve.test.ts     # Unit tests
      DayN.vue          # UI component
  composables/
    useInputLoader.ts   # For loading input files
```

## Testing
- Run tests: `npm run test:run`
- Run tests in watch mode: `npm run test`
- Tests should be fast and deterministic
- Avoid console output in production code (it creates noise in test output)

## Input Files
- **IMPORTANT**: Place puzzle inputs in `public/DayNInput.txt` (e.g., `Day1Input.txt`, `Day7Input.txt`)
- Use the exact naming pattern: `Day` + number + `Input.txt` (no hyphens, capital D, capital I)
- Input files are loaded via fetch using `useInputLoader`
- In your Vue component, use: `const { input } = useInputLoader('Day7Input.txt')`
- DO NOT include folder paths in the filename - files in `public/` are served from the root

## Code Style
- Follow existing TypeScript patterns
- Use meaningful variable names
- Keep functions focused and testable
- Prefer pure functions for solution logic
